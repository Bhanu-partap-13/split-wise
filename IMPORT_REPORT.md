## How This Report Is Shown in the App

1. **Step 1 — Upload**: User drops `expenses_export.csv` into the dropzone.
2. **Step 2 — Validate**: This report is rendered in the UI. Each row is color-coded:
   - 🟢 **Green** = clean
   - 🟡 **Yellow** = warning, will be imported with a flag
   - 🔴 **Red** = error, will be skipped
   - Individual issue details are shown inline on hover/expand
3. **Step 3 — Confirm**: User clicks "Import X Rows" — only after reviewing this report.
   Skipped rows are never written to the database.
